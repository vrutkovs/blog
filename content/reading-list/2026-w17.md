# Artificial Intelligence & Society

* [Exclusive: Microsoft To Shift GitHub Copilot Users To Token-Based Billing, Tighten Rate Limits](https://www.wheresyoured.at/news-microsoft-to-shift-github-copilot-users-to-token-based-billing-reduce-rate-limits-2/)

  Leaked internal documents show Microsoft planning to shift GitHub Copilot from per-request to token-based billing, suspend new individual and student signups, and remove Anthropic Opus from cheaper subscription tiers. The change is driven by near-doubled weekly operating costs since January, marking the end of subsidized AI coding assistants.

  Tags: Microsoft, GitHub, Copilot, Billing, Cost

* [Is Claude Code going to cost $100/month? Probably not—it's all very confusing](https://simonwillison.net/2026/Apr/22/claude-code-confusion/#they-reversed-it)

  Anthropic quietly updated its pricing page to restrict Claude Code to $100/month+ plans — no announcement, no warning — sparking immediate backlash from users and educators who had relied on $20/month Pro access.   Tags: Anthropic, Pricing, Claude, Experiment, Trust

* [High-Quality Chaos](https://daniel.haxx.se/blog/2026/04/22/high-quality-chaos/)

  Open-source security has entered a "high-quality chaos" era: AI tools have doubled the volume of bug bounty submissions while maintaining the same confirmed vulnerability rate as pre-AI reports. The curl project expects a record number of CVEs in 2026, and the maintainer burden of triaging this new volume is becoming unsustainable across the ecosystem.

  Tags: AI, Security, Vulnerability, Curl, Chaos

* [Relicensing with AI-assisted rewrite](https://tuananh.net/2026/03/05/relicensing-with-ai-assisted-rewrite/)

  The `chardet` project relicensed from LGPL to MIT after using Claude Code to rewrite its codebase — a move that sparked legal debate over whether AI-assisted rewrites satisfy the "clean room" requirement when the AI was trained on and exposed to the original code. A recent Supreme Court ruling requiring "Human Authorship" for copyright creates an additional paradox: if the rewritten code isn't copyrightable, the MIT license itself may be invalid.

  Tags: License, AI, Copyright, Rewrite, Chardet

* [FinOps for Kubernetes: when "it works" isn't enough](https://jorijn.com/en/blog/finops-for-kubernetes-when-it-works-isnt-enough/)

  Average Kubernetes CPU utilization sits at just 10%, with the remainder wasted on over-provisioned resource requests the scheduler allocates structurally. * [how a roblox cheat and one AI tool brought down vercel's entire platform](https://webmatrices.com/post/how-a-roblox-cheat-and-one-ai-tool-brought-down-vercel-s-entire-platform)

  A Roblox cheat downloaded by a Context.ai employee exposed a supply chain attack path: Lumma Stealer grabbed session cookies, which were used to pivot into a Vercel employee's corporate Google account via an AI tool with broad OAuth permissions. "Non-sensitive" Vercel environment variables — encrypted at rest but decryptable from the dashboard backend — were exfiltrated and offered for $2 million.

  Tags: Security, Vercel, Breach, AI, OAuth

* [This Week in Kubernetes Development - Week Ending April 19, 2026](https://lwkd.info/2026/20260423)

  Kubernetes v1.36 ships with fine-grained kubelet API authorization, MutatingAdmissionPolicy, and Workload Aware Scheduling. Other highlights include the KMM operator v2.6.0, a new mandatory SHA pinning policy for GitHub Actions, and KEP-5538 advancing secure CSI driver service account token handling.

  Tags: Kubernetes, Release, Security, Feature, Storage

* [go.sum Is Not a Lockfile](https://words.filippo.io/gosum/?source=Mastodon)

  `go.sum` is a local cache for the Go Checksum Database — it serves a security purpose but has no effect on version resolution. `go.mod` is Go's actual lockfile, listing every exact direct and transitive dependency since Go 1.17, making the separate manifest/lockfile patterns common in other ecosystems unnecessary.

  Tags: Go, Module, Dependency, Lockfile, Checksum

* [Whale Fall](https://nesbitt.io/2026/02/21/whale-fall.html)

  Abandoned open source projects create succession ecosystems analogous to whale falls: initial scavenger forks, then opportunist projects reusing interfaces, then protocols and formats that support specialized communities for decades. License changes trigger immediate community forks; corporate consolidation, by contrast, tends to prevent these ecosystems from forming at all.

  Tags: OpenSource, Ecosystem, Project, Fork, Succession
