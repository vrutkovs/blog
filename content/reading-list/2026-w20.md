# Read This Now

* [Kubernetes 1.36 quietly buried one of the worst RBAC patterns in production](https://jorijn.com/en/blog/kubernetes-1-36-fine-grained-kubelet-authz-buries-nodes-proxy/)

  The `nodes/proxy` RBAC pattern — commonly granted to monitoring tools — allowed privilege escalation and remote code execution via a plain GET request, bypassing audit logs and admission control. KEP-2862 in 1.36 replaces it with eight specific subresources (`nodes/metrics`, `nodes/log`, etc.), enabling least-privilege access for observability without handing over the full kubelet API. Migration requires manual auditing of existing ClusterRoles.

  Tags: Kubernetes, Security, RBAC, Kubelet, Migration

# Artificial Intelligence & Society

* [You Need AI That Reduces Maintenance Costs](https://www.jamesshore.com/v2/blog/2026/you-need-ai-that-reduces-your-maintenance-costs)

  AI coding agents that double output without reducing maintenance costs produce a temporary productivity spike followed by permanent decline. For sustainable benefit, AI must decrease maintenance costs by the inverse of its code output rate — otherwise teams end up locked in a cycle of diminishing returns.

  Tags: AI, Productivity, Maintenance, Cost, Code

# Software Engineering & Cloud

* [Lies, damned lies, and Elastic's benchmarks](https://www.gouthamve.dev/lies-damned-lies-and-elastics-benchmarks/)

  Elastic's "30x faster than Prometheus" blog post misrepresents Prometheus storage usage, withholds benchmarking code, and produces non-reproducible results. Custom tests show Prometheus ingesting millions of samples per second while Elastic slows, times out, and would need over 40 hours to complete the same workload. A call for standardized, reproducible metrics benchmarks and maintainer collaboration before publishing such claims.

  Tags: Benchmark, Elastic, Prometheus, Performance, Metric

* [Language Registries Are Unstable by Default](https://nesbitt.io/2026/05/15/language-registries-are-unstable-by-default.html)

  Language package registries publish new versions immediately with no promotion gates — the same risk profile Debian explicitly labels "unstable." This design offloads integration and security burden to individual consumers and drives recurring supply chain vulnerabilities. Client-side cooldowns and private proxies are reinventing fragments of distribution release processes. Registry-level stable channels, modeled on distro practices, are what's actually needed.

  Tags: Package, Registry, Stability, Security, Supplychain
