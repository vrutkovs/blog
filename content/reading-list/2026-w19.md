# Artificial Intelligence & Society

  Richard Dawkins wrote a piece describing falling for Claude and concluding it was conscious. Rebecca Watson picks apart the bad Turing Test reasoning, the rhetorical self-flattery, and the odd choice to assign female pronouns to an AI while publicly opposing trans women — using *Solaris* as a frame for why humans project onto non-human intelligence.

  Tags: Dawkins, Chatbot, AI, Consciousness, Ego

* [Premium: How OpenAI Kills Oracle](https://www.wheresyoured.at/how-openai-kills-oracle/)

  The "Stargate" AI infrastructure project is largely theater: Oracle is piling on massive debt to build data centers for an OpenAI that lacks capital to sustain payments. GPU technology is aging fast, Oracle's core businesses are stagnating, and the math points to one outcome.

  Tags: Oracle, OpenAI, Stargate, AI, Finance

* [Running AI agents safely in a microVM using docker sandbox](https://andrewlock.net/running-ai-agents-safely-in-a-microvm-using-docker-sandbox/)

  How to run AI coding agents in "dangerously-skip-permissions" mode without burning your machine: `sbx` tool setup, network policy configuration, sandbox creation, and Git repo management. microVM-based isolation provides real security boundaries; commit signing and performance remain pain points.

  Tags: AI, Sandbox, Docker, Security, Agent

* [There has to be a way](https://www.bloodinthemachine.com/p/there-has-to-be-a-way)

* [How OpenClaw Got Safer in Public](https://openclaw.ai/blog/openclaw-security-in-public/)

A flood of mostly-invalid security reports preceded a more structured approach; contributions from NVIDIA, Microsoft, GitHub, and others proved crucial.

# Software Engineering & Cloud

* [Weekend at Bernie's](https://nesbitt.io/2026/05/08/weekend-at-bernies.html)

  A significant share of critical open-source packages are unmaintained — dead, dormant, or untested for responsiveness. With AI-assisted vulnerability discovery accelerating disclosure rates, the absence of maintainers to issue fixes becomes a security crisis compounded by registries that have no clear process for handling dead packages.

  Tags: Package, Security, Maintenance, OpenSource, Vulnerability

* [A GitHub for maintainers](https://nesbitt.io/2026/05/02/a-github-for-maintainers.html)

  A wishlist for forges that put inter-project coordination first: downstream dependency testing, dependent feeds, fork network visualization, package registry insights, a shared package cache for safer CI, and renaming "issues" to "tickets." Features aimed at the maintainer managing a dependency graph, not the solo developer pushing commits.

  Tags: Maintainer, Dependency, Forge, CI, Package

* [Before GitHub](https://lucumr.pocoo.org/2026/4/28/before-github/)

  Armin Ronacher on open source project hosting from self-managed servers and SourceForge through GitHub's rise and perceived decline. GitHub democratized collaboration and became a critical archive, but also enabled the micro-dependency explosion. A case for decentralization — and for publicly-funded archival that survives any commercial platform.

  Tags: GitHub, OpenSource, Dependency, Archive, Platform

* [NVME is not a hard disk](https://blog.koehntopp.info/2021/05/25/nvme-is-not-a-hard-disk.html)

  NVMe storage has fundamentally different failure characteristics from spinning disks — the redundancy assumptions baked into traditional data center designs don't apply, requiring a rethink of how databases and networked storage are architected.

  Tags: Storage, Network, Database, Redundancy, DataCenter

* [How Hard Is It To Open a File?](https://blog.sebastianwick.net/posts/how-hard-is-it-to-open-a-file/)

  Opening files across privilege boundaries is a TOCTOU minefield. Path strings are inherently unsafe references; file descriptors are the safe primitive. `libglnx` and `glnx_chaseat` offer FD-based traversal that avoids symlink races — a gap in POSIX, GLib, and Rust APIs that already produced a critical Flatpak sandbox escape.

  Tags: Security, Filesystem, Vulnerability, FileDescriptor, Flatpak

* [The lopsided economics of vulnerabilities](https://opensourcesecurity.io/2026/05-vulnerability-economics/)

  Vulnerability disclosure has shifted from coordinated company-mediated processes to direct reporting against open source projects. Companies want to suppress, researchers want credit, open source maintainers just want a fix. The era of security embargoes is ending — leaving maintainers with sudden undisclosed drops and no preparation time.

  Tags: Vulnerability, OpenSource, Security, Disclosure, Company

* [Zed is 1.0](https://zed.dev/blog/zed-1-0)

  Zed hits 1.0: built from scratch in Rust with a custom GPUI framework, explicitly rejecting Electron. Full language support, Git integration, debugging, and AI-native features including parallel agents and edit predictions ship at launch. Zed for Business launches alongside, with DeltaDB for human-AI collaboration on the roadmap.

  Tags: Editor, Performance, AI, Rust, Collaboration
